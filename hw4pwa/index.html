The Characteristics of a Progressive Web Application (PWA)

A Progressive Web App, or PWA, is a website that can act and look like a native app- like the best of both worlds. PWAs begin with an HTML page and likely include some CSS and JavaScript, but have additional features that allow them to have some of the benefits of native apps. For example, the ability to send push notifications to the user, provide offline access, or give users the option to open the app in their browser or install onto their home screen with its own icon and UI. Additionally, unlike native apps (which need dedicated development teams for different platforms like iOS or Android) one PWA can work on different operating systems.

There are three main technical aspects of a PWA that allow all of this to happen. The first is a secure connection served over HTTPS. Second is service workers. Service workers is an API that intercepts outgoing network requests, allowing PWAs to execute offline capability and better integration with the device (which permits for native-like features). Last is a web app manifest, which is a JSON file required for PWAs. A web app manifest controls how the app will look to the user and the information that allows the browser to install the PWA. These components work together so that a PWA can be: reliable regardless of network status, quick, and possibly more engaging than a traditional website or native app could be on their own!

To see more about PWAs, visit this guide from Mozilla. Learn how PWAs are built, and you can even try building one yourself.
